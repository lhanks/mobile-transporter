# MT-104: File List/Download REST API

## Description

Expose endpoints for listing, downloading, and deleting files.

## Acceptance Criteria

- [ ] `GET /api/files` returns JSON array of file metadata
- [ ] `GET /api/files/:id` serves file with proper headers
- [ ] `DELETE /api/files/:id` removes file and returns 204
- [ ] Returns 404 for non-existent file ID
- [ ] Content-Type header set correctly for downloads
- [ ] Content-Disposition header triggers browser download

## Technical Notes

### Endpoints

```
GET /api/files
Response 200:
[
  { "id": "uuid1", "name": "file1.pdf", "size": 12345, "uploadedAt": "..." },
  { "id": "uuid2", "name": "file2.jpg", "size": 67890, "uploadedAt": "..." }
]

GET /api/files/:id
Response 200: File stream with headers
Content-Type: application/octet-stream (or detected mime type)
Content-Disposition: attachment; filename="original-name.pdf"

Response 404:
{ "error": "File not found" }

DELETE /api/files/:id
Response 204: (empty body)

Response 404:
{ "error": "File not found" }
```

### File Streaming

Use `res.sendFile()` or `fs.createReadStream().pipe(res)` for efficient file delivery.
