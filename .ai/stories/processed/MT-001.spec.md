# Mobile Transporter - Project Specification

## Overview

Mobile Transporter is a cross-platform file sharing application that enables seamless file transfers between devices through a shared folder system. Users can drop files into a shared folder, and all connected devices can see and download those files.

## Problem Statement

Transferring files between devices (phone to computer, computer to tablet, etc.) often requires:
- Third-party cloud services with size limits
- USB cables and manual file management
- Email attachments with size restrictions
- Complex network share setups

Mobile Transporter solves this by providing a simple, local-network file sharing solution.

## Target Platforms

- **Web** - Browser-based access (primary interface)
- **Mobile** - iOS and Android via PWA or native app
- **Desktop** - Windows, macOS, Linux via Electron or web

## Core Features

### 1. Shared Folder
- Central location where files are stored temporarily
- All connected devices can view folder contents
- Files persist until manually deleted or auto-expire

### 2. File Upload
- Drag-and-drop file upload
- Multi-file selection support
- Upload progress indicator
- Support for any file type

### 3. File Download
- Browse available files
- Single-click download
- Batch download support
- Download progress tracking

### 4. Device Discovery
- Automatic discovery of devices on same network
- Manual connection via code/link
- Device naming for easy identification

### 5. Real-time Updates
- Live file list updates when files added/removed
- Connection status indicators
- Transfer notifications

## Technical Architecture

### Technology Stack

| Component | Technology | Rationale |

| Frontend | TypeScript + React | Type safety, component reusability |

| Build | Vite | Fast development, optimized builds |

## API Design

### REST Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/files` | List all shared files |
| POST | `/api/files` | Upload a file |
| GET | `/api/files/:id` | Download a file |
| DELETE | `/api/files/:id` | Remove a file |
| GET | `/api/status` | Server health check |

### WebSocket Events

| Event | Direction | Description |
|-------|-----------|-------------|
| `file:added` | Server → Client | New file uploaded |
| `file:removed` | Server → Client | File deleted |
| `device:joined` | Server → Client | New device connected |
| `device:left` | Server → Client | Device disconnected |

## Security Considerations

1. **Local Network Only** - Initially operates on local network only
2. **No Authentication** - Trust-based for simplicity (v1)
3. **File Validation** - Basic file type/size validation
4. **Auto-expiry** - Files auto-delete after configurable period

## Future Enhancements (Out of Scope for v1)

- End-to-end encryption
- QR code pairing
- Cloud relay for remote access
- File preview (images, PDFs)
- Transfer history
- Mobile native apps

## Success Criteria

- [ ] Files can be uploaded from any device
- [ ] Files appear on all connected devices in real-time
- [ ] Files can be downloaded from any device
- [ ] Works on web, mobile browsers, and desktop browsers
- [ ] Simple, intuitive UI requiring no instructions
