# MT-102: File Storage Service

## Description

Implement server-side file storage with basic CRUD operations.

## Acceptance Criteria

- [ ] `uploads/` directory created on server startup
- [ ] FileMetadata interface defined: `{id, name, size, uploadedAt}`
- [ ] In-memory Map to store file metadata
- [ ] FileService class implemented with all methods
- [ ] Files saved to `./uploads/{uuid}-{originalName}`
- [ ] Can list/get/delete files programmatically

## Technical Notes

### FileMetadata Interface

```typescript
interface FileMetadata {
  id: string;           // UUID
  name: string;         // Original filename
  size: number;         // Size in bytes
  uploadedAt: Date;     // Upload timestamp
}
```

### FileService Methods

```typescript
class FileService {
  saveFile(file: Express.Multer.File): FileMetadata
  listFiles(): FileMetadata[]
  getFile(id: string): { path: string; metadata: FileMetadata } | null
  deleteFile(id: string): boolean
}
```

### Storage Pattern

- Files stored as: `./uploads/{uuid}-{originalFilename}`
- Metadata stored in Map<string, FileMetadata>
- UUID prevents filename collisions
