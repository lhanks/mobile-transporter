# MT-111: Dark Mode Support

## Description

Add dark mode to the application with a toggle in the header. The app should remember the user's preference and respect system-level dark mode settings.

## Acceptance Criteria

- [ ] Dark mode toggle button in header
- [ ] All UI components styled for dark mode
- [ ] Preference persisted in localStorage
- [ ] Respects system prefers-color-scheme on initial load
- [ ] Smooth transition between modes

## Technical Notes

- Use Tailwind's dark mode feature with class strategy
- Store preference in localStorage under 'theme' key
- Add dark: variants to all components
- Consider using a React context for theme state

## Files to Modify

- `src/client/App.tsx` - Add theme context provider
- `src/client/components/Header.tsx` - Add dark mode toggle
- `src/client/components/*.tsx` - Add dark: variants
- `tailwind.config.js` - Enable dark mode class strategy
- `src/client/index.css` - Add dark mode base styles

## Testing

- [ ] Manual testing: toggle works correctly
- [ ] Manual testing: preference persists across page reloads
- [ ] Manual testing: respects system preference on first visit
