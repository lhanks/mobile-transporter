# MT-112: Vercel Deployment

## Description

Configure the project for deployment to Vercel with automatic deployments triggered by pushes to the main branch. Deployments should only succeed if all unit and e2e tests pass.

## Acceptance Criteria

- [ ] vercel.json configuration file created
- [ ] Build command configured for monorepo structure
- [ ] GitHub Actions workflow for CI/CD
- [ ] Tests must pass before deployment proceeds
- [ ] Environment variables documented

## Technical Notes

- Vercel serverless functions for the Express backend
- Build outputs: client to dist/client, server to api/
- Use Vercel CLI or GitHub integration
- GitHub Actions workflow runs tests before allowing merge/deploy

## Files to Modify

- `vercel.json` - Create Vercel configuration
- `.github/workflows/ci.yml` - Create CI workflow with tests
- `package.json` - Add vercel-specific scripts if needed
- `README.md` - Document deployment process

## Testing

- [ ] CI workflow runs on push/PR
- [ ] Tests execute successfully in CI
- [ ] Vercel preview deployments work
- [ ] Production deployment succeeds
